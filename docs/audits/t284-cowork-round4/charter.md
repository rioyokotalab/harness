# Charter

## Task

Dogfood the round-three `wait-copilot` and adaptive focused-check changes in a
real Codex-driver/Claude-co-pilot successor session. Measure whether one bounded
wait replaces manual status polling, audit its final JSON and exit semantics,
and challenge the publication candidate for safety or usability regressions.

## Boundaries

Codex is driver and sole potential target mutator after reconciliation; Claude
receives only sealed direct-child stages. This round begins as a read-only
acceptance audit and may implement only a narrowly frozen defect found by real
dogfooding. No settings, credentials, packages, remotes, services, external
messages, or cleanup. The owner's ongoing three-hour instruction is the go for
an evidence-supported repository-only fix.

`wait-copilot` call with the native wrapper PID. Require one parseable final
JSON object, correct advisory/non-authorizing labels, fresh full conjunction,
and unchanged protected/stage/seal inputs before import. Reciprocal critique
must see actual native/wait elapsed results and identify any race, API, prompt,
or adaptive-check defect. Complete with receipts, focused checks, fresh-clone
validation, and no untracked residue.
